DATA DICTIONARY
===============

EXTERNAL ENTITIES:
------------------
1. User (Students/Staff)
   - Description: Individuals who report lost or found items
   - Input: Registration data, item details, login credentials
   - Output: Match notifications, status updates, contact information

2. Admin (Staff/Management)
   - Description: System administrators who review and approve matches
   - Input: Match approval/rejection decisions
   - Output: Status updates, system management actions

PROCESSES:
----------
1. User Management Process
   - Function: Handle user registration, login, and authentication
   - Input: User registration data, login credentials
   - Output: User accounts, session information

2. Item Reporting Process
   - Function: Process lost and found item submissions
   - Input: Item details, descriptions, locations, dates
   - Output: Item records in database

3. Matching Algorithm Process
   - Function: Automatically identify potential matches
   - Input: Lost and found item data
   - Output: Potential match records with pending status

4. Admin Review Process
   - Function: Review and approve/reject potential matches
   - Input: Pending match records
   - Output: Approved/rejected match status

5. User Notification Process
   - Function: Notify users of match status changes
   - Input: Match status updates
   - Output: Dashboard updates, contact information access

DATA STORES:
------------
1. Users Database
   - Content: User account information
   - Fields: id, fullname, email, phone, password, created_at

2. Lost Items Database
   - Content: Lost item reports
   - Fields: id, user_id, item_name, description, date_lost, location

3. Found Items Database
   - Content: Found item reports
   - Fields: id, user_id, item_name, description, date_found, location

4. Matched Items Database
   - Content: Match records with status
   - Fields: id, lost_item_id, found_item_id, status, matched_on

DATA FLOWS:
-----------
1. User Input Flows:
   - Registration data → User Management Process
   - Login credentials → Authentication Process
   - Item details → Item Reporting Process

2. System Output Flows:
   - Match notifications → User Dashboard
   - Status updates → User Interface
   - Contact information → Approved Users

3. Admin Input Flows:
   - Match decisions → Admin Review Process
   - System management → Administrative Interface

4. Database Operations:
   - CRUD operations on all tables
   - Data validation and integrity checks
   - Transaction management

KEY DATA FLOW CHARACTERISTICS:
-----------------------------
- Bidirectional: Data flows both ways between users and system
- Real-time: Immediate processing of user inputs
- Automated: Matching algorithm runs automatically
- Secure: Authentication and authorization at each step
- Auditable: All actions are logged and trackable
